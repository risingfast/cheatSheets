Tips and Tricks
    Do not use leading spaces in python for formatting - Python code structure is based on indenting 
        Code blocks are indented (normally 4 chars). Indent tabs must convert to spaces
    Clear the screen with import os ... os.system('cls')  # on windows, os.system('clear') on linux
    Don't name your program files the same as package names

Installation 
    Linux
        The executable for Python is at /usr/bin/python 
    Windows
        C:\Users\gjarman\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Python 3.7  -- private install
        C:\Program Files (x86)\Python37-32 -- shared install
    Installation on Apache2 Webserver on Linux for Python and mod_Python
        Python is installed as a cgi on Linux2. 
        It is possible to also install as a mod_python module by editing httpd.conf
        With this standard cgi installation, any script with a python shebang line (#! python3) will invoke the python enterpreter
        The script should be placed somewhere under the /srv/www/cgi-bin/ filesystem root or top
        Normally a subdirectory under cgi-bin is used for this (ie /srv/www/cgi-bin/python/<script.py>
        The http address via the apache server for this loction is http://<host>:port/cgi-bin/python/<script.py>
        Installing as mod_python, requires a pointer to the mod_python.so file (the DSO object)
        mod_python.so is located at /usr/lib/apache/mod_python.so on the Suse 9.1 linux configuration
        python must be an entry in the apache sysconfig file (see /etc/sysconfig/apache2) to load mod_python

    Indentation: 4 spaces
    Max line width: 79 chars
    Docstring width: 72 chars
    Line continuation: wrap in parentheses, brackets or braces (use \ as a last resort
    Use lower_case_underscores in variable_names and not camelCase style variableNames

Data Types
    Primitives
        Integers 1, 2, 3, 44 (division of integers yields floats)
        Floating Point Numbers 1.1, 2.2, 3.3, 4.567, -8.9
        Decimal
        Fraction
        Booleans: True, False (converted to 1 and 0 in arithmetic expressions)
            None, Zero empty sequences and empty dictionaries evaluate to False
        Complex (using j for imaginary component) e.g. x = complex(1, 2)
    Containers
        Strings -- single quoted. 
        Lists -- compound type ... my_list = [1, 2, 3, 4] for element in my_list: print(element)
        Tuples -- immutable lists using () or no brackets e.g. x = ('a', 'b') or x = 'a', 'b' are identical
        Sets -- unordered lists without duplicates using {} notation
        Dictionaries
        None -- contains only None value (similare to Null) and the return value of any function w/out an explicit return value

    Tips
        Paths must use '/' and not '\' on MS DOS machines
        Files are iterable objects and can be read on line at a time
    Text file modes are 
        'r' read a file (default mode if not specified)
        'r+' read and write an existing file. The stream is positioned at the beginning of the file 
        'w' write a file only. Over-write (truncate an existing file even if nothing is written. Otherwise creates a new file
        'w+' read and write a file. Create if not existing. The stream is positioned at the beginning of the file. 
        'a' append a file, 
        'a+' append or read a file and create new if it does not exist 
        'x' exclusive create
    Binary file modes are (Windows only, suppressed adding the \r character to each \n) 
        'rb' -- read binary
        'r+b' -- read and write an existing file
        'wb' - overwrite binary 
        'w+b'-- overwrite existing binary or create new file if it does not exist 
        'ab' -- append binary
        'a+b'-- append binary or create a new file if it does not exist 
        'xb'
    File Mode Reference from https://stackoverflow.com/questions/16208206/confused-by-python-file-mode-w/16212401
        r Opens a file for reading only. The file pointer is placed at the beginning of the file. This is the default mode.
        rU Read in universal mode. Automatically translates newlines to \n in any operting system environment
        r+ Opens a file for both reading and writing. The file pointer will be at the beginning of the file.
        rb+ Opens a file for both reading and writing in binary format. The file pointer will be at the beginning of the file.
        w Opens a file for writing only. Overwrites the file if the file exists. If the file does not exist, creates a new file for writing.
        wb Opens a file for writing only in binary format. Overwrites the file if the file exists. If the file does not exist, creates a new file for writing.
        w+ Opens a file for both writing and reading. Overwrites the existing file if the file exists. If the file does not exist, creates a new file for reading and writing.
        wb+ Opens a file for both writing and reading in binary format. Overwrites the existing file if the file exists. If the file does not exist, creates a new file for reading and writing.
        a Opens a file for appending. The file pointer is at the end of the file if the file exists. That is, the file is in the append mode. If the file does not exist, it creates a new file for writing.
        ab Opens a file for appending in binary format. The file pointer is at the end of the file if the file exists. That is, the file is in the append mode. If the file does not exist, it creates a new file for writing.
        a+ Opens a file for both appending and reading. The file pointer is at the end of the file if the file exists. The file opens in the append mode. If the file does not exist, it creates a new file for reading and writing.
        ab+ Opens a file for both appending and reading in binary format. The file pointer is at the end of the file if the file exists. The file opens in the append mode. If the file does not exist, it creates a new file for reading and writing.

Data Structures

Lists
    Lists are sequences
    list1 = [] or list = list() creates a null list of objects
    list1 = list(<object>) -- constructor converts an interable object (tuple, string, set or dictionary) to a list
    Lists can have duplicates
    Lists are mutable and Tuples are not. Otehrwise they are the same
    list = ['a', 'b', 'c'] -- defines a mutable list as a value that can contain mutiple values in an ordered sequence
        list is a reference or pointer to the list so changing elements in list will change any copies
    for element in list: print(list) -- prints all the members of a list
    Join elements into a string with array_str = "".join(list) -- use a different value in "" to insert between each value
    Elements in a list are referenced by number -- List[4]. The first element is zero
    Lists within a list are referenced by two numbers -- List[4][3] is the 4th element in the 5th element of the list
    The base index is 0 in a list
    The index in a list can only be integer, not float
    Negative indices start on the end of a list and count backards. -1 is the end, -2 is second from end etc

Tuples
    Tuples are sequences

            Use nested () for dimensional tuples ... myTuple = (('a', 'b'), ('c, 'd')) etc
            Use indices to reference elements ... for value in myTuple: value[-1] ... gives last element in each value group
    A list is a value that can be passed to a function
    Lists are defined as [item1, item2, item3, item4...] ... a comma-separated list of items

Conditionals, Flow Control and Loops
    There is no switch case statement in Python so use if elif else ...
    Flow Control statements comprise and condition (expression) and a clause (the block of code if the condition is true)
    Conditions evaluate to a boolean value: True or False
    Clauses begin with indentation and can contain other blocks
    Clauses end when indentation ends
    if flow control
        if <var> == <value>:      # if is required but elsif and else are optional
            <if statements>
        elif <condition>          # there can be multiple elif's but only one or none will be executed
            <elif statements>
        else:                     # optional else
            <else statements>
    for loop (2 forms)
        for <var> in range(<start>, <stop>, <step>):        # <stop> is required and is greater than max value, <start> defaults to 0, <step> defaults to 1
            <statements>                                    # iteration occurs up to <stop> but not including <stop>
            break                 # to force a break out of the loop or an infinite loop
            continue              # to force the next run of the loop w/out running intervening code
        for <var> in <iterable list>                         # loops for each object in list
            <statements>
            break
            continue
    while loop
        while <condition>:        # runs while as long as condition evaluates True
            <statements>
            break                 # to force a break out of the loop or an infinite loop
            continue              # to force to the next run of the loop w/out running intervening code

Scopes and Namespaces
    LEGB: Local, Enclosing, Global and Built-in
        Local - variables defined within a function
        Enclosing -  variables defined within the local scope of an enclosing function
        Global - top level of a module or explicitly declared
        Built-in - pre-assigned names -- infrequently used
    global x -- defines a global x variable (within a function)
    nonlocal x -- defines a nonlocal x in enclosing functions
    A namepace is a mapping from names to object
    The script read by the interpreter is __main__
    Built-in methods are in the namespace 'builtins'
    Scope is a textual region where the namespace is directly accessible (using unqualified references)
    Global scope is created when a program begins and there is only one global scope
    Local scope is created when a function is called and destroyed when a function returns a value
    Local scope can access global variables but cannot access variables in any other local scope
    You can use the same name for different variables in different local scopes
    Global variables can be read from the local scope so avoid using local variables named the same as globals
    Use the global statement to set or modify a global variable within a local scope
