Tips and Tricks and Troubleshooting Issues
    Don't use float in an elemet style because the element floats outside it's container <div>
    Cross Origin Resource Sharing (CORS) - set http response header Access-Control-Allow-Origin: * on server side in Ajax response or CGI
    https://en.wikipedia.org/wiki/Cross-origin_resource_sharing
    Dont add () to callbacks because they will cause immediate execution

HTML Tag and Element Structure
    Two main areas: head and body
        head contains only metadata
        body contains viewable data elements
    <!DOCTYPE html> -- the only DOCTYPE declaration for HTML5. Declaration to the browser identifying the html version to render
    <html lang="en-US"> -- html tag with embedded lang attribute (for search engines)
        <head> -- metadata region (notrendered). Metadata typically define the document title, character set, styles, scripts, and other meta information)
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <meta name="author" content="Geoffrey Jarman">
            <meta name="description" content="This is the description"> -- picked up by the search engine
            <title>This is the page title</title> -- appears in the page tab of the browser
             <link rel=""icon" href=""html5.png" type=""image/x-icon"> -- add an icon to the page tab
        </head>
        <body> -- defines the body area of HTML containing non-metadata (rendered data)
            <h1>Main page title</h1>
            <h2>Subtitle</h2>
            <h3>Section Title</h3>

            
        </body>
    </html>
    Attributes are within the opening HTML tag and define characteristics of the tag (metadata)
    Empty elements have no content or ending tag. They have attributes (eg <img>).
    Meaningless elements <DIV> and <SPAN> are used to assign CSS but are otherwise meaningless

Inline vs Block Elements
    Block elements appear (e.g. <h1> <p> <div>
    Inline elements do not start on a new line and only take up as much width as necessary ... they appear immediately infront of and after their content (e.g. <SPAN>)

HTML Rules
    Extra spaces are removed (aka whitespace collapsing) by rendering (space, tab, newline) unless protected by <pre> preformat tag which preserves whitespace
    Case doesn't matter in tags but lower case is the standard
    Quotes can be single our double on attributes and nesting is allowed on opposite types
