URL Taxonomy
    https://www.example.site.com/services/summary?arg1=value&arg2=value2
    Protocol: https://
    Subdomain: example
    Domain: site
    TLD (Top Level Domain): .com
    Subfolder: /services -- a directory on the host
    Slug: /summary -- a document on the host with default .html ending
    ? -- first get parameter separator
    & -- further get parameters (multiple allowed

HTTP Response Codes
    200 -- OK, content delivered
    400 -- Bad request
    403 -- Forbidden
    404 -- Not found
    500 -- Internal server error (consult the errorlog)

HTML Tag and Element Structure
    <!DOCTYPE html> -- the only DOCTYPE declaration for HTML5. Declaration to the browser identifying the html version to render
    <html lang="en-US"> -- html tag with embedded lang attribute (for search engines)
        <head> -- metadata region (notrendered). Metadata typically define the document title, character set, styles, scripts, and other meta information)
            <title>This is the page title</title> -- appears in the page tab of the browser
            <meta name="description" content="This is the description"> -- picked up by the search engine
            <meta charset="UTF-8">
        </head>
        <body>
            defines the body area of HTML containing non-metadata (rendered data)
        </body>
    </html>
    Attributes are within the opening HTML tag and define characteristics of the tag (metadata)
    Empty elements have no content or ending tag. They have attributes (eg <img>).
    Meaningless elements <DIV> and <SPAN> are used to assign CSS but are otherwise meaningless
    Block elements appear around their content (e.g. <DIV>
    Inline elements appear infront of and after their content (e.g. <SPAN>)

JS GET Requests
    GET requests encode all information into the URL
    The format is <URL>?param1=<value1>&param2=<value2>&param3=<value3> ...
    The parameter string is saved as the QUERY_STRING environment variable in the server, starting after the ? separator

JS Fetch and promise using .then or async and await (in Ajax Calls)
    Fetch returns a promise
    Promises are handled with .then() or async await syntax e.g. fetch('rainbow.jpg').then(response => {return response.blob();});
        .then can be chained for successive handling, and chained to .catch for errors
    Example
        async function start() {
            const response = await fetch("https://dog.ceo/api/breeds/list/all");  -- fetch is async and must include 'await' in it's call
            const jsonData = await response.json();                               -- response is converted to json
            console.log(jsonData);                                                -- extract the json data from the response
        start();                                                                  -- call the async function which also returns a response 
        start().catch(error => {                                                  -- or call the async function with a .catch if it returns an error response
            console.log(error);
        });
